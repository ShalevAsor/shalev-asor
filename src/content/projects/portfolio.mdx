## Overview

Most developer portfolios are boring - just a list of projects with brief descriptions. I wanted something better. This portfolio turns project documentation into an experience with custom MDX components, interactive examples, and detailed technical breakdowns.

**Why this approach?** Most portfolios say "I built an e-commerce site with Next.js and Stripe." Cool, but how? This portfolio explains the technical challenges, architectural decisions, and interesting solutions. It's a portfolio that actually demonstrates technical depth.

## How It Works

<Collapse title="Velite + MDX Pipeline">

**1. Content Management**

Projects live in `src/content/projects/*.mdx`. Velite processes them at build time:

```typescript
// velite.config.mts
export default defineConfig({
  collections: {
    projects: {
      pattern: "projects/**/*.mdx",
      schema: s.object({
        title: s.string(),
        slug: s.slug("projects"),
        description: s.string(),
        date: s.isodate(),
        published: s.boolean(),
        featured: s.boolean(),
        tags: s.array(s.string()),
        github: s.string().url().optional(),
        demo: s.string().url().optional(),
        // ... more fields
      }),
    },
  },
});
```

Velite generates:

- TypeScript types in `.velite/index.d.ts`
- JSON data in `.velite/projects.json`
- Automatic slug validation and permalink generation

**2. MDX Processing**

Each MDX file goes through rehype plugins:

- **rehype-slug** - Adds IDs to headings
- **rehype-pretty-code** - Syntax highlighting with One Dark Pro theme
- **rehype-autolink-headings** - Makes headings clickable

**3. Component Mapping**

Custom components are mapped to MDX in `MDXContent.tsx`:

```typescript
const mdxComponents = {
  Callout,
  Collapse,
  TechStack,
  Challenge,
  // ... 20+ custom components

  // Override default HTML
  h2: (props) => (
    <h2 className="text-3xl font-semibold mb-4 mt-12" {...props} />
  ),
  code: (props) => (
    <code className="bg-muted px-1.5 py-0.5 rounded" {...props} />
  ),
};
```

**4. Static Generation**

Next.js generates static pages at build time:

- `/` - Homepage
- `/projects` - Project listing
- `/projects/[slug]` - Individual project pages

Result: Fast, SEO-friendly, no runtime processing.

## Design Decisions

<FeatureGrid>
  <FeatureCard title="Velite over Contentlayer">
    Velite is lighter, faster, and actively maintained. Contentlayer was
    abandoned. Velite's API is cleaner and TypeScript support is better.
  </FeatureCard>
  <FeatureCard title="Custom Components over Templates">
    Building components once means every project page is consistent. Adding a
    new project is just writing MDX - no styling or layout decisions.
  </FeatureCard>
  <FeatureCard title="Static Generation">
    All projects are static HTML at build time. Fast page loads, great SEO,
    works with JavaScript disabled. Perfect for a portfolio.
  </FeatureCard>
  <FeatureCard title="One Dark Pro Theme">
    Developers recognize this theme from VS Code. Syntax highlighting feels
    familiar and professional.
  </FeatureCard>
  <FeatureCard title="No Database">
    Content is markdown files in git. Changes are tracked in version control. No
    CMS needed. Simpler deployment.
  </FeatureCard>
  <FeatureCard title="Server Components">
    Everything is a Server Component by default. Client components only for
    interactivity (modals, carousels). Better performance.
  </FeatureCard>
</FeatureGrid>

## The UserFlowGrouped Story

<Collapse title="Why I Built This Component">

**The Problem:**

The original `UserFlow` component displayed flows vertically with arrows between steps. For projects with 20-30 steps (like Tale By You's checkout flow), it was way too long. Users had to scroll forever.

**First Solution - Horizontal Timeline:**

I created `UserFlowTimeline` with horizontal scrolling cards. Better, but still not ideal for very long flows.

**Final Solution - UserFlowGrouped:**

Group steps into collapsible phases:

```tsx
<UserFlowGrouped
  phases={[
    {
      title: "Template Selection",
      icon: "ðŸ“",
      steps: [
        "Browse templates",
        "Select template",
        // ... more steps
      ],
    },
    {
      title: "AI Generation",
      icon: "ðŸŽ¨",
      steps: [
        "Generate images",
        "Process results",
        // ... more steps
      ],
    },
  ]}
/>
```

Users see 4-5 phase headers. Click to expand details. **Much more compact** while still showing the complete flow.

This is a perfect example of iterating on portfolio components based on real needs.
